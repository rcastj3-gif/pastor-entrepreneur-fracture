## DEPLOYMENT PATH (5 STEPS)

### 1️⃣ PUSH TO GITHUB (10 minutes)
```bash
cd /tmp/pastor-entrepreneur-fracture

# Initialize git
git init
git add .
git commit -m "Initial: Pastor-Entrepreneur Fracture landing page"

# Create repo on GitHub
# https://github.com/new
# Name: pastor-entrepreneur-fracture

git remote add origin https://github.com/YOUR_USERNAME/pastor-entrepreneur-fracture.git
git branch -M main
git push -u origin main
```

### 2️⃣ DEPLOY TO VERCEL (2 minutes)
- Go to https://vercel.com/new
- Select your GitHub repo
- Click "Deploy"
- **Your site is LIVE** 🎉

### 3️⃣ CONFIGURE INTEGRATIONS (5 minutes)
In Vercel Dashboard → Settings → Environment Variables:
```
GRAWT_API_KEY = your_key_from_grawt.io
AGENTMAIL_API_KEY = your_key_from_agentmail.to
```

### 4️⃣ CONFIGURE ANALYTICS (5 minutes)
Update Google Analytics ID in `index.html`:
```html
<script async src="https://www.googletagmanager.com/gtag/js?id=G-YOUR_ID"></script>
```

### 5️⃣ TEST & LAUNCH (10 minutes)
- Visit your live site (vercel auto-assigns URL)
- Test form submission
- Check email received
- Monitor analytics firing
- Share with audience

**Total time: ~30 minutes from GitHub to production**
